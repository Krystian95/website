Jolie allows for the development of distributed applications, thus communication among services is a key aspect to be considered when building a system with Jolie. An interface is the Jolie artifact which permits to declare all the functionalities, called operations exhibited by a service. Every message exchange is fulfilled by means of an operation.

Operations

> Go on top

An operation is a functionality exposed by a service which can be invoked by other applications. Whenever you want to invoke another service, you must do so by invoking one of its operations, and vice versa (whenever another application wants to invoke yours, it must do so by invoking an operation of your program). There are two kinds of operations:

  • One-Way: it simply receives a message;
  • Request-Response: it receives a message and then sends back a response.

Interface declaration can be enriched with message types declaration. OneWay messages require only one message type whereas RequestResponse operations require both the request and the response type. Left type declaration si the request type whereas the right one refers to the response one.
Some examples about interface declarations with message types can be found at section Types.

Interfaces - RequestResponse fault declaration

> Go on top

RequestResponse operations can raise faults as a reply message. Faults must be declared within the interface, they could be simple faults without any type associated or they can be typed faults.
Some examples about interface declarations withh faults can be found at section Fault handling.
