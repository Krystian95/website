This tool allows for the creation of a WSDL document starting from a Jolie interface.

  • filename.ol: it is the jolie service file whose inputPort must be transformed in a soap one
  • soapInputPortName: it is the name of the soap port to be exposed as a Web Service
  • output_file: the file where storing the generated WSDL document (default is MyWsdl.wsdl)
It is worth noting that the protocol defined into the inputPort to be converted must be set to soap.

Jolie interface guidelines

In order to correctly transform a Jolie interface into a WSDL document the following guidelines must be followed when programming the interface:
  • Native types in operation declaration are not permitted. For example, the following declaration is not avoided:
    interface MyInterface { RequestResponse: ----myOp( string )( int ) }
  • All the complex types must have void value in the root. The following declarations are not permitted:
    type Type1: int { ----.msg: string } type Type2: string { ----.msg: string } interface MyInterface { RequestResponse: ----myOp( Type1 )( Type2 ) }
