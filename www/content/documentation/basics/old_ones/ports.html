Ports

As seen in section Interfaces, services can communicate with each other in order to obtain a composition. Programs get more complex by composing messages; thus, a fundamental notion for programming with Jolie is to understand how to define services and how make them communicate with each other.

In Jolie every communication is fulfilled by means of ports and the language defines two kind of ports:

  • inputPorts: used for receiving messages from other services;
  • outputPorts: used for sending messages to other services.

inputPorts

An input port is defined by four parameters:

  • a name, unambiguously identifying it among other inputPorts;
  • a location, defining the communication medium to use;
  • a protocol, specifying the protocol to be used to communicate (in section protocols it's available a list of protocols supported by Jolie);
  • a list of interfaces (optional) to deploy with the input port;

outputPorts

In the same way inputPorts allow the specification of an input point, outputPorts define endpoints able to send messages to a correspondent inputPort.

Like inputPorts, outputPorts are defined by the same four parameters (name, location, protocol and an optional list of interfaces), but in this case they're used to specify the parameters for sending the message.
