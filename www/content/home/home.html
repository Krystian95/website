Crystalyzing Microservices programming concepts
Jolie is the first language which crystalizes microservices programming concepts. Programmers and system designers can focus on services and their functionalities without taking care of their deployment. Program once, deploy more!
No differences between remote services and local services
In Jolie there is no difference between local services and remote services. You can build a monolithical applications as it is a microservice application. If you decide to distribute a part of it, just take it and execute it in another machine.

Protocol independence: Jolie services are ESB free!
Your Jolie services can receive messages by using a protocol and then send a message by using another protocol. You don't need ESB framework and you can customize your own protocols.
Structured architectural primitives
Jolie offers primitives for determining the structure of your software architecture: Aggregation, Redirection, and Embedding. They allow you to design architectures within service's code.
Revolutionize the way you develop web applications
In the Jolie world web GUIs are completely independent and technologically decoupled from the microservice system. A special few lines of code service, called Leonardo, acts as a web server which can interoperate with web applications written in different technologies. In Jolie there is native support for GWT, JS and JSON.
No frameworks, just an interpreter!
You do not need special frameworks for deploying services, just install the Java based interpreter of Jolie and run the services you write. You can run a service in single line command!
