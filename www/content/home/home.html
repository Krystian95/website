Crystalyzing Microservices programming concepts
Jolie is the first language which crystalizes microservices programming concepts. Programmers and system designers can focus on services and their functionalities without taking care of their deployment.
Distribution, reusability and orchestration are achieved by design. No functions, no objects but services!
No differences between remote services and local services
In Jolie there is no difference between local services and remote services. You can build a monolithical applications as it is a microservice application. If you decide to distribute a part of it, just take it and execute it in another machine.

Protocol independence!
Your Jolie services can receive messages by using a protocol and then send a message by using another protocol. You don't need special frameworks and you can customize your own protocols.
Structured primitives both for behaviour and architetcures!
Jolie offers primitives for determining the structure of your software architecture: Aggregation, Redirection, and Embedding. They allow you to design architectures within service's code.
Orchestration: in Jolie there are no differences among services and orchestrators. A service is always an orchestrator and viceversa.
Aggregation: Aggregation allows to compose the API of the existing services into a new one just connecting services.
Redirection: Redirection allows to create services which act as proxy towards other services.
Embedding: Embedding allows to run services into other services as they are inner components or children processes.
Termination, compensation and dynamic error handling are powerful means for dealing with faults!
Error handlers can be updated dynamically depending on the executed activities thanks to the install primitive. Termination and compensation handlers facilitate the design of orchestrators with inner parallel activities.
Revolutionize the way you develop web applications
In Jolie interfaces are clearly defined, thus web GUIs are completely independent and technologically decoupled from the microservice system. A special few lines of code service, called Leonardo, acts as a web server which can interoperate with web applications written in different technologies. In Jolie there is native support for GWT, JS and JSON.

You don't need frameworks, just an interpreter!
You do not need special frameworks for deploying services, nor ESB for achieving integration. You just install the Java based interpreter of Jolie and run the services you write. You can run a service in single line command!
You can start to build your architecture just now with few lines of code!
