Communication primitives

  • one-way
  • request-response
  • notification
  • solicit-response
  • Example


When interfaces and ports are defined, it is possible to achieve communication between two or more services. There are four types of communication primitives:

one-way

> Go on top

It waits for receiving a message and it stores it into the specified variable.

request-response

> Go on top

The semantics of the request-response is:
  • it waits for receiving a message;
  • when the message is received, it stores its content into the request_var;
  • it executes the body_activities
  • it sends the response message whose content is that of response_var

notification

> Go on top

It sends a message whose content is stored in send_var at the outputPort op_name. It is worth noting that the outputPort must be defined before sending a message to it.

solicit-response

> Go on top

The solicit-response sends a message whose contents is defined in send_var to the outputPort op_name. Then it waits for the response from the invoked service. When the response is received it stores the message in recv_var.
